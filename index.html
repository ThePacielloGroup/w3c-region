w3c-region example

An eXperiMental custom element to use in place of headingless (no h1-h6) article/section elements.

w3c-region is available to skewer on github.

Semantics: div too little, article/section too much, w3c-region just right.

In cases where an author wants to identify a region of page content in the code (which may include multiple section or article elements), but does not want to add a heading, what element should they use? Wrapping them in a div is viewed with disdain because div's are not semantic enough. Wrapping them in an article or section adds unwanted/un-needed outline semantics. Thats where w3c-region comes in. It provides a semantic wrapper without effecting the outline.

Plain Old Skool

<div>

The div element has no special meaning at all.

HTML 5.1

Old Skool semanticized

<div class="region">

... It can be used with the class, lang, and title attributes to mark up semantics common to a group of consecutive elements.

HTML 5.1

Middle Skool semanticized

<div role="region">

From representing nothing to anybody except those who can read the attribute scratchings, to representing a peceivable unit via the magic of ARIA.

OldNew Skool semanticized

<div is="w3c-region">

Old skool CSS style with semantic underpants.

Brand New Skool semanticized

<w3c-r>

Slick webcomponent leisure suit with discrete semantic under garment.

Usage

is="w3c-region"

<div is="w3c-region">
<section>
<h2>Old Skool</h2>
<p><code>&lt;div class=&quot;region&quot;&gt;</code></p>
</section>
<section>
<h2>Middle Skool</h2>
<p><code>&lt;div role=&quot;region&quot;&gt;</code></p>
</section>
</div>

<w3c-r>

<w3c-r>
<section>
<h2>OldNew Skool</h2>
<p><code>&lt;div is=&quot;w3c-region&quot;&gt;</code></p>
</section>
<section>
<h2>Brand New Skool</h2>
<p><code>&lt;w3c-r&gt;</code></p>
</section>
</w3c-r>
